# RepoLens User Growth Strategy

## Objective

Increase adoption of RepoLens among students, open-source beginners, and developers by making repository onboarding faster and easier using AI.

## Week-wise Plan

### Week 1

* Share RepoLens with classmates and internship peers.
* Collect at least 25 user reviews using a Google Form.
* Fix usability issues identified from feedback.

### Week 2

* Publish a detailed README with examples.
* Share the project on LinkedIn and GitHub.
* Demonstrate RepoLens during technical club meetings and workshops.

### Week 3

* Add support for additional repositories and improve analysis quality.
* Improve multilingual support.
* Enhance the local AI (Ollama) workflow.

### Week 4

* Release updated versions based on user feedback.
* Improve onboarding experience.
* Expand documentation and tutorials.

## User Feedback Process

1. Collect feedback through Google Forms.
2. Categorize issues into bugs, feature requests, and UI improvements.
3. Create GitHub/GitLab issues.
4. Prioritize high-impact improvements.
5. Release updates and document changes in the changelog.

## Long-term Vision

RepoLens aims to become an AI-powered onboarding assistant for open-source communities, helping new contributors understand repositories, reduce onboarding time, and improve participation in collaborative software development.
