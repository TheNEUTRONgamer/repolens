# RepoLens Constitution

## 1. Purpose

RepoLens is an open-source onboarding assistant designed to help developers understand unfamiliar repositories and identify contribution opportunities. Our goal is to lower the barrier to open-source participation.

## 2. Guiding Principles

*   **Simplicity:** The tool should be intuitive and easy to use.
*   **Clarity:** Generated reports must be clear, concise, and actionable.
*   **Modularity:** The codebase will be modular to encourage extensibility and maintainability.
*   **Community-Driven:** We welcome and encourage community contributions to improve the tool.

## 3. Coding Standards

*   **Language:** Python 3.11+
*   **Formatting:** Code will be formatted using `ruff format`.
*   **Linting:** Code will be checked with `ruff` to ensure quality and consistency.
*   **Dependencies:** Project dependencies are managed with `uv`.

## 4. Quality Assurance

*   **Testing:** All new functionality should be accompanied by unit tests.
*   **Pre-Commit Hooks:** Pre-commit hooks will be used to automate checks before commits.
*   **Manual Verification:** All features will be manually verified to ensure they meet the specified requirements.

## 5. Documentation

*   The `README.md` will serve as the primary entry point for users and contributors.
*   Specifications and plans will be maintained in the `/specs` directory.
